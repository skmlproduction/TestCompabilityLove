# Google Play — Data safety (черновик ответов)

Сверьте с [официальной формой](https://support.google.com/googleplay/android-developer/answer/10787469) перед отправкой. Текст в приложении: `assets/legal/data_collection.html`.

## Сбор данных

| Вопрос | MVP ответ |
|--------|-----------|
| Собираете ли данные? | **Да** (минимум: покупки через Google; опционально реклама) |
| Все данные шифруются в транзите? | **Да** для сетевых вызовов (HTTPS); имена **не** уходят на наш сервер |
| Можно запросить удаление? | Имена — **Настройки → Очистить сохранённые имена**; аккаунт Play — через Google |

## Типы данных (типично для MVP)

| Тип | Собирается | Обязательно | Цель | Передача третьим лицам |
|-----|------------|-------------|------|------------------------|
| Имя (ввод в тестах, incl. love protocol) | Да, локально | Нет | Функция приложения | Нет (наш сервер не получает) |
| Покупки в приложении | Да | Да (если Premium) | Подписка/разовая покупка | Google Play |
| Device/Advertising ID | Только если ads + consent | Нет в MVP без AdMob | Реклама | Партнёры (v2) |

## Практика в приложении

- Кэш последних имён: DataStore, **исключён из backup** (`backup_rules.xml`).
- Очистка: `settings_clear_saved_names`.
- Analytics / Crashlytics: **нет** в MVP — в форме не отмечать.
